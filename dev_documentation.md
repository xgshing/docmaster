# 软件开发文档

## 1. 平台架构与技术选型

### 1.1 平台架构
- **前端**：原生桌面应用（支持 Windows/macOS/Linux）
- **后端**：Python + MySQL
- **文档编辑内核**：OnlyOffice 开源版（自行部署）
- **云存储**：腾讯云 COS
- **部署环境**：腾讯云轻量服务器（开发与测试）→ 腾讯云（生产环境）
- **桌面端工程形态**：Electron + Vue3 + TS 单仓；

### 1.2 技术选型
| 层级 | 技术栈 |
|------|--------|
| 桌面应用 | Electron + Vue3 + TypeScript |
| 后端 API | Python + Django |
| 数据库 | MySQL |
| 文档编辑 | OnlyOffice（自部署） |
| 云存储 | 腾讯云 COS |
| 通信协议 | HTTPS |

## 2. 登录模块

### 功能
- 用户登录与角色验证
- 支持多角色登录，确保系统的权限管理

### 细节
- 登录界面支持用户名、密码输入
- **角色定义**：
  - **超级管理员**：具有最高权限，能够创建、删除、修改管理员账号，查看所有文档，管理所有用户，拥有所有共享文件夹的完全控制权（删除、修改权限）
  - **普通管理员**：可以创建/删除自己创建的分组，设置用户分组，创建共享文件夹并设置权限
  - **普通用户**：只能查看和编辑个人文档库中的文件，以及根据权限访问共享空间
- **登录互踢机制**：同一账号不允许多客户端同时在线。当同一账号在新设备登录时，原设备会被强制踢下线，并提示“您的账号已在其他设备登录，当前会话已结束”。登录“互踢”机制具体要求是：只校验 API 请求时发现 session 失效并提示退出。

## 3. 用户与角色管理模块

### 功能
- 用户管理、角色和权限设置

### 细节
- **超级管理员**：
  - 可以创建/删除/修改管理员账号
  - 可以为普通用户注册账号（普通用户账号由超级管理员注册提供，用户无法自主注册）
  - 可以修改用户名和管理员名
  - 可以重置所有用户的密码
  - 可以修改自己的用户名和密码（需输入当前密码验证身份）
  - 密码忘记时，通过预留邮箱验证重置
- **普通管理员**：
  - 不能管理普通用户账号
  - 可以创建/删除自己创建的分组
  - 可以为用户分配分组
  - 如果删除的分组中还有成员，成员将不再显示该分组
  - 分组分配 ≠ 用户账号管理，仅可分配分组关系，无权操作用户账号本身
- **普通用户**：仅能操作个人文档库和访问有权限的共享空间，无法自主注册账号、无法自主修改个人资料；账号密码仅可由超级管理员重置，账号启用/禁用权限仅归属超级管理员
- **密码规则**：统一8位长度即可，无额外复杂度限制

## 4. 分组管理模块

### 功能
- 为用户分配分组、管理分组

### 细节
- **分组创建**：超级管理员和普通管理员都可以创建分组
- **分组使用**：所有管理员创建的分组，其他管理员都可以使用（如设置共享文件夹权限时选择）
- **分组删除**：
  - 超级管理员可以删除任意管理员创建的分组
  - 普通管理员仅能删除自己创建的分组
- **分组查看**：用户可以查看分组由哪个管理员创建
- **多分组分配**：用户可以同时属于多个分组
- **分组权限**：管理员创建共享文件夹时，可以设置每个分组的文件访问权限（查看/编辑）
- **分组删除影响**：删除的分组中还有成员时，成员将不再显示该分组
- **权限优先级**：如果一个用户/分组同时拥有多个权限（如通过不同分组继承），以**最高权限**为准

## 5. 文档管理模块

### 功能
- 文档上传、查看、编辑、删除、共享、保存、打印、导出等

### 细节

#### 5.1 个人文档库
- **存储位置**：个人文档库显示的文件均为本地电脑储存，可在软件中自定义文件目录
- **跨设备说明**：如果在其他电脑登录同一账号，无法显示个人文档库内容（因为个人文档库关联的是当前电脑本地储存的文档）
- **文件操作**：
  - 支持将本地电脑的文件/文件夹**添加**到个人文档库的指定目录下（支持拖拽添加）
  - 支持在软件中创建目录（在软件中创建目录是真实创建本地文件系统目录），便于文件归类
  - 编辑保存后直接保存到本地
  - 可以查看文件在本地电脑的路径
- **删除操作**：个人文档库文件/文件夹的删除，本地电脑文件也会被同步删除，并移入本地电脑回收站；
- **本地联动规则**：
  - 若电脑本地原文件被删除，软件内个人文档库对应文件也同步删除
  - **同步机制**：软件启动时自动扫描个人文档库配置的本地目录，与软件内索引进行比对；同时支持**实时监听**，本地文件变化时自动同步更新软件内索引
- **恢复方式**：通过“导入”功能重新添加本地文件

#### 5.2 共享空间
- **共享文件夹**：由管理员创建，可设置用户/分组的访问权限（查看/编辑）
- **管理员目录权限**：文件夹创建者（管理员）可以在共享文件夹内新建子文件夹，支持搭建多层目录结构
- **文件操作**：
  - 用户可以查看/复制有权限的共享文件
  - 有编辑权限的用户可以直接编辑云端文件（不自动产生本地副本）
  - 如需离线编辑，可手动“添加到个人文档库”
- **文件夹权限继承**：文件夹内的文档会继承文件夹的权限

#### 5.3 支持的文件类型
| 文件类型 | 在线编辑 | 在线查看 | 发布到共享空间 | 打印 |
|----------|----------|----------|----------------|------|
| Word | ✅ | ✅ | ✅ | ✅ |
| Excel | ✅ | ✅ | ✅ | ✅ |
| PPT | ✅ | ✅ | ✅ | ✅ |
| PDF | ❌ | ✅ | ✅ | ✅ |
| 图片格式（JPG/PNG/JPEG等） | ❌ | ✅ | ✅ | ✅ |

- **补充说明**：
  - 图片文件支持在线预览、缩放查看
  - 系统不存在加密文档
  - 仅拥有查看权限的用户，同样支持文件下载、导出、打印

#### 5.4 文件发布到共享空间
- 用户可以把个人文档库中的文件或文件夹**发布**到管理员创建的共享文件夹中
- **权限要求**：仅对共享文件夹有**编辑权限**的用户才能执行发布操作
- **发布机制**：发布到共享空间的文件是**独立副本**，与个人文档库中的原文件没有关联。用户在本地删除或移动原文件，不会影响共享空间中已发布的文件
- **版本更新**：用户可以通过重新发布操作来更新共享空间中的文件版本
- **同名文件处理**：检测到同名文件时，按第5.8节规则处理

#### 5.5 文件删除（共享空间）
- **软删除机制**：共享空间的文档删除后进入回收站
- **回收站功能**：
  - 文件保留期限为**30天**，过期后自动彻底删除
  - 显示回收站文件占用的存储空间大小
  - 彻底删除时，同时清理腾讯云 COS 上的物理文件
- **回收站权限**：
  - 超级管理员：查看全平台所有回收站文件，可恢复、彻底删除任意文件
  - 普通管理员：仅能查看自己创建的共享文件夹中被删除的文件，可恢复、彻底删除对应文件
  - 普通用户：无回收站访问权限
- **覆盖操作说明**：当用户覆盖共享区同名文件时，被覆盖的原文件自动移入回收站

#### 5.6 文件导出/下载
- **从共享空间下载**：支持将共享空间的文件/文件夹下载到本地电脑

#### 5.7 文件信息显示
- 不管是共享空间还是个人文档库的文件，都需要显示**文件最后一次编辑的时间**
- 共享空间文件还需显示**最后一次编辑的用户名**

#### 5.8 同名文件处理规则
- **查重范围**：同名文件全局共享空间检索判定，遍历全部共享空间（含归档文件）查重
- **权限要求**：仅对共享文件夹有**编辑权限**的用户才能执行覆盖操作
- **检测到同名文件时**，弹窗提示用户选择：
  - **重命名**：系统自动在文件名或文件夹名后添加当前日期（格式：文件名_YYYYMMDD.扩展名）
  - **覆盖**：用新文件替换原有文件（需二次弹窗确认），被覆盖的原文件自动移入回收站
  - **取消**：放弃本次操作
- **特殊情况**：如果已存在包含日期后缀的同名文件（如“报告_20260101.docx”），则仅提供**覆盖**和**取消**选项，不再提供重命名
- **自带日期后缀文件规则**：新上传文件本身自带日期后缀时，若全局共享空间存在同名文件，执行覆盖操作时，原共享区同名文件自动移入回收站并重命名；用户重复发布多次带日期后缀文件时，所有被覆盖的原文件均可无限移入回收站
- **回收站移入文件命名规则**：原文件名 + 移入日期 + 移入时间

## 6. 文档编辑模块

### 功能
- 支持在线编辑 Excel、Word、PPT 文档
- 支持保存、导出、打印等功能

### 细节

#### 6.1 编辑功能
- 右侧为文档编辑区域，集成 OnlyOffice 文档编辑功能
- 仅 **Word、Excel、PPT** 支持在线编辑
- PDF 和图片格式仅支持查看，不支持编辑和保存
- **编辑权限定义**：编辑权限仅允许修改文件内容，不允许对文件/文件夹进行结构性操作（新建、重命名、移动、删除）。如需创建新文件，应先在个人文档库中创建，再发布到共享空间
- **编辑临时文件缓存**：存放于对应文件所在目录

#### 6.2 编辑锁机制（防冲突）
- 当用户编辑文档时，**锁定文档**
- 其他用户只能以**只读方式**打开
- 编辑时显示**正在编辑的用户名**
- **锁无自动释放机制**：编辑锁不会自动超时释放，由管理员手动管理
- **管理员强制解锁**：
  - 超级管理员和普通管理员均可强制解锁被锁定的文档
  - 解锁后，原有编辑会话直接失效，原先锁定文档的用户刷新页面后即可恢复正常状态，不再出现占用拦截提示

#### 6.3 保存功能
- **在线保存**：文档编辑完成后，在线保存至服务器（共享空间）
- **本地保存**：用户可以选择将文件保存到本地（个人文档库）
- **PDF导出**：支持导出文档为 PDF 格式

#### 6.4 打印功能
- **打印预览**：支持打印预览，查看文档打印效果
- **直接打印**：可以直接将文档发送至打印机进行打印

## 7. 文件存储与共享模块

### 功能
- 管理文件的上传、下载、共享和权限设置

### 细节

#### 7.1 共享文件夹创建与管理
- **普通管理员**：可以创建共享文件夹，可在自身共享目录内新建子文件夹
- **超级管理员**：可以创建共享文件夹，拥有所有共享文件夹的完全控制权，可删除、修改任意管理员创建的共享文件夹及内部内容（不包含转移所有权功能）
- **创建者权限**：
  - 创建共享文件夹的管理员拥有该文件夹的完全管理权限
  - 可以对该文件夹及里面的文件进行**删除、移动、编辑**
  - 可以设置哪些用户/分组可以查看该文件夹
  - 可以设置哪些用户/分组可以编辑该文件夹内的文件
  - 可以选用其他管理员创建的分组配置权限
- **文件/文件夹移动**：创建者可以将文件/文件夹移动到**自己创建的其他共享文件夹**中。移动后，权限继承新文件夹的权限设置

#### 7.2 共享文件夹权限设置
- **权限类型**：
  - **查看权限**：只能查看/复制文件，不能编辑
  - **编辑权限**：可以编辑文件内容（不能进行结构性操作）
- **权限范围**：
  - 可以针对**单个用户**设置权限
  - 可以针对**分组**设置权限
  - 文件夹权限**继承**给子文件夹和文件
- **权限优先级**：直接指定用户权限 > 分组权限 > 父文件夹继承权限

#### 7.3 共享空间操作权限总结
| 操作 | 创建该文件夹的管理员 | 有编辑权限的用户 | 仅有查看权限的用户 |
|------|---------------------|------------------|---------------------|
| 查看文件 | ✅ | ✅ | ✅ |
| 编辑文件内容 | ✅ | ✅ | ❌ |
| 发布/覆盖文件 | ✅ | ✅ | ❌ |
| 新建文件 | ❌* | ❌ | ❌ |
| 新建子文件夹 | ✅ | ❌ | ❌ |
| 重命名 | ✅ | ❌ | ❌ |
| 移动文件/文件夹 | ✅（仅限到自己创建的其他共享文件夹） | ❌ | ❌ |
| 删除文件/文件夹 | ✅ | ❌ | ❌ |
| 设置权限 | ✅ | ❌ | ❌ |
| 归档 | ✅ | ✅ | ✅ |

> *注：新建文件需先在个人文档库中创建，再发布到共享空间

#### 7.4 共享文档添加到个人文档库
- 用户可以通过以下方式**将共享文档/文件夹添加到个人文档库**：
  - **右键点击**或选中文件/文件夹后，选择“添加到个人文档库”
  - 系统弹出本地目录选择器，确认后将文件自动复制到本地个人文档库
  - **拖拽操作**：将共享空间的文件直接拖拽到左侧个人文档库的目录下，系统自动下载并保存

#### 7.5 归档功能
- **归档/取消归档**：所有权限用户均可对共享空间的文件/文件夹进行归档操作
- **归档状态同步**：归档状态与用户账号绑定，同一用户在不同设备登录时，归档视图状态保持一致。归档操作不影响其他用户对该文件的查看和操作
- 归档后的文件/文件夹**默认不显示**
- 用户可以主动进入“**归档视图**”查看和恢复已归档内容

#### 7.6 共享历史记录
- 系统记录每个文档和文件夹的共享历史
- 记录内容：共享者、被共享者、共享时间等

#### 7.7 文件访问记录
- 系统记录文件的访问历史
- 记录内容：访问者、访问时间、操作内容等

### 细节
- 界面设计需符合产品级设计标准

## 10. 安全与权限控制

### 功能
- 确保数据的安全性和权限的严格管理

### 细节
- 所有敏感数据通过 **HTTPS** 协议进行加密传输
- **用户权限控制**：系统根据用户角色和分组设定严格的访问权限
- **数据安全**：用户密码、文档等敏感信息加密存储
- **审计日志**：系统记录所有重要操作（登录、上传文档、共享文档、删除文件等）
- **覆盖操作确认**：覆盖操作需要二次弹窗提醒，防止误操作
- **日志统一留存时长**：全系统所有操作日志、访问日志、共享日志保留**1年**，到期自动清理
- **账号登录规则**：同一账号**不允许多客户端同时在线**。当同一账号在新设备登录时，原设备会被**强制踢下线**，并提示“您的账号已在其他设备登录，当前会话已结束”

## 11. 数据库设计

### 功能
- 数据存储与管理

### 细节
- **数据库选择**：MySQL
- **核心表设计**：

| 表名 | 主要字段 | 说明 |
|------|----------|------|
| 用户表 | 用户ID、用户名、密码、角色、邮箱、最后登录IP、最后登录时间、账号状态（启用/禁用）、当前登录会话标识 | 存储用户基础信息，支持登录互踢 |
| 文档表 | 文档ID、文件名、文件类型、文件大小、云端COS存储路径、所属空间类型、最后编辑时间、最后编辑用户、归档状态 | 统一存储共享云端文件元数据 |
| 本地索引表 | 索引ID、用户ID、本地文件路径、文件名、文件大小、关联状态 | 独立存储个人文档库本地文件索引，与云端文件物理拆分 |
| 分组表 | 分组ID、分组名、创建者ID、创建者角色、父分组ID | 存储分组信息，记录创建管理员身份 |
| 用户分组关联表 | 用户ID、分组ID | 用户与分组的多对多关联关系 |
| 共享文件夹表 | 文件夹ID、文件夹名称、创建者ID、创建时间、父文件夹ID、存储路径、归档状态 | 支持多层目录结构 |
| 共享权限表 | 权限ID、文件夹ID、用户ID、分组ID、权限类型（查看/编辑） | 存储文件夹针对用户、分组的权限配置 |

## 14. 总结

本项目是一个**以本地文件为核心的桌面文档管理软件**，具备以下特点：

- **本地优先**：个人文档库基于本地文件系统，编辑保存均在本地；删除个人文档库文件/文件夹时，同步删除本地文件并移入本地回收站
- **跨设备说明**：个人文档库关联当前电脑本地储存，其他电脑登录无法显示个人文档库内容；共享空间和归档状态支持跨设备同步
- **账号安全**：同一账号不允许多客户端同时在线，新设备登录时原设备被强制踢下线
- **云端协作**：共享空间提供团队协作能力，支持精细的角色、分组权限管控，权限叠加统一取最高权限
- **全局最高权限**：超级管理员拥有全平台所有资源最高权限，可管理分组、共享文件夹、用户、回收站全部内容
- **安全可靠**：HTTPS加密、文档独占编辑锁、管理员强制解锁、软删除+回收站30天自动清理、完善日志留存1年
- **功能完整**：支持 Word/Excel/PPT/PDF/图片的查看、在线编辑、预览缩放、打印、导出、下载、多级目录、个人归档
- **角色清晰**：超级管理员、普通管理员、普通用户三级严格权限体系，分组分配与用户账号管理边界独立
- **规则闭环**：全局同名文件查重（跨所有共享文件夹全局唯一）、回收站命名规范、会话机制、账号登录互踢策略、缓存路径全部统一明确无冲突
